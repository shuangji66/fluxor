# 订阅工作模式

在「订阅配置」页面中，Fluxor 允许您在两种不同类型的订阅加载逻辑间切换，满足不同用户的多机场管理需求。

---

## 融合模式 (Merge)

当您在页面上切换至融合模式并保存时，后台程序的合并逻辑如下：

### 1. 节点打散与合并
系统会并发向列表中所有启用的机场链接发起拉取。下载完成后，将每个订阅中的所有富强节点提取出来，剔除重名节点，最终混合并入同一个总的节点池中。

多订阅可开启节点前缀以区分不同订阅来源的节点

### 2. 生成统一的策略组
系统会加载统一的规则分流模板（标准 或 详细），并将上述合并后的所有节点写入到预设的策略组（如 节点选择、谷歌服务、苹果服务 等）中。

标准模版：根据国内外 IP 和 网址进行分流。
详细模版：根据ruleset规则进行分组分流，满足场景分流需求。

### 优势与适用场景
* **优势**：各机场节点互补。如果其中一个机场因网络波动出现断连，您可以直接在网页的同一策略组内一键切到另一个机场的节点上，体验极其连贯。
* **适用场景**：适合手头拥有多个低价机场并希望自动整合为一个配置文件的用户。

---

## 切换模式 (Switch)

当您在页面上切换至切换模式并保存时，后台的运行逻辑将发生变化：

### 1. 物理隔离与独立激活
每个机场订阅的数据都是独立的。在网页上，所有添加的订阅以独立卡片排列。当您点击某个卡片时，它会亮起蓝色边框，代表被当前激活（Active）。

### 2. 独立配置文件渲染
点击「保存并应用」后，系统只会加载这**唯一一个**被激活订阅的节点，并且直接提取并沿用该订阅源自身所自带的内置策略组和分流规则（而不是使用面板预置的 标准 或 详细 模板）。其他未选中的订阅节点文件将只作为备份存放在 `/var/apps/Fluxor/shares/Fluxor/proxies/` 目录下，不加载运行。

### 优势与适用场景
* **优势**：完美保持机场原装分流逻辑与策略树，各机场间物理隔离，互不干扰，性能损耗最少。
* **适用场景**：适合购买了不同用途的高级专线机场（例如一个只用于外贸查资料，一个用于游戏加速），需要在不同场景间彻底切换隔离的用户。
