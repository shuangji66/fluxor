# Fluxor 用户使用手册

本手册专门为初学者量身定制，通过简单的步骤引导您从零开始使用 Fluxor 面板管理您的网络代理。同时，手册中所有的默认端口、文件路径及规则逻辑均与系统源代码完全一致，确保内容准确写实。

---

## 快速上手三步走

如果您是第一次使用 Fluxor 面板，请直接按照以下三个步骤操作，即可完成网络代理的配置与使用：

### 第一步：导入订阅并激活保存
1. 在面板导航中进入「订阅配置」页面。
2. 在页面中间找到「订阅列表」，点击「添加订阅」按钮。
3. 在弹出的输入框中填写信息：
   * **名称**：为您机场的节点起一个名字（例如 my_airport）。名称必须是唯一的，不能与已有订阅重名。
   * **订阅链接**：粘贴您在机场购买并复制的 Clash 或 Mihomo 订阅 URL。
   * **检测间隔**：设置定时自动更新订阅的时间（以秒为单位，默认 86400 秒即 24 小时。设为 0 代表关闭自动更新，仅手动更新）。
   * **健康检查间隔**：设置后台节点自动测速的频率（以秒为单位，默认 300 秒即 5 分钟）。
   * **节点名前缀过滤**：如果您只想保留特定地区的节点，可以在此输入过滤关键字（例如输入 阿里云，则只有名称包含该字符的节点会被筛选保留；留空则导入所有节点）。
4. 在上方参数区，选择您需要的分流规则集：
   * **基础版 (Base)**：适合大部分普通用户，包含常见的国内外服务及基本分流规则。
   * **完整版 (Full)**：分类极其详尽的规则策略，适合对网络分流有细致要求的进阶用户。
   *(注意：新版面板不再提供精简版 Lite 选项，请在 Base 和 Full 之间根据需要选择。)*
5. 选择订阅模式：
   * **融合模式 (Merge)**：将您添加的所有机场订阅里的节点混合在一起使用，适合同时拥有多个机场并希望自动选优的用户。
   * **切换模式 (Switch)**：每次只启用一个选中的订阅。卡片周围出现蓝色边框并高亮显示时，代表该订阅被成功激活（Active）。
6. 设置面板控制密钥（Secret）与端口，默认混合代理端口为 7890，控制端口为 9090。
7. 点击页面最下方的「保存并应用」按钮。系统会在后台将端口、密钥及订阅数据写入 fluxor.json 配置文件，并自动合并节点与规则集模板，在后台生成 config.yaml 配置文件并引导内核运行。

### 第二步：测试节点延迟与一键切换
1. 进入「代理管理」页面。
2. 此时，您将看到刚才保存生成的所有策略组（如 节点选择、谷歌服务、苹果服务 等）。点击任意策略组卡片可折叠或展开它下属的代理节点。
3. 点击页面顶部的「测试全部延迟」按钮，或者点击单个策略组右上角的闪电图标，系统会开始批量测速。
   *(为防止浏览器假死，面板在后台代码中强制限制了最大同时测速并发数为 10，测速较多时请耐心等待排队。)*
4. 根据延迟测试结果，节点右侧会显示具体的延迟数值并用颜色进行标注：
   * **绿色 (延迟 150ms 以下)**：代表连接状况极佳，推荐选择。
   * **黄色 (延迟 300ms 以下)**：代表连接一般，可以正常使用。
   * **红色 (延迟 300ms 以上或显示超时)**：代表网络连接较差，可能会有明显卡顿。
5. 点击您满意的节点卡片，边框亮起后即表示该策略组已成功切换到此节点。

### 第三步：开启全局接管 (TUN) 与透明代理 (TProxy)
如果要在您的主机或路由设备上接管所有流量，需要配置全局流量捕获。
1. 进入「内核配置」页面。
2. **TUN 虚拟网卡**：在 TUN 栏中将开关打开。引擎（Stack）可以选择 System（系统自带）或 gVisor、Mixed。启用 TUN 模式可直接在系统层接管所有网络流量。
3. **TProxy 透明代理**：如果使用透明代理，请确保 TProxy 端口不为 0（可在配置页中修改），然后打开 TProxy 开关。此外，您可以点击「TProxy例外列表」弹窗修改排除哪些局域网 IP（源例外）或目标 IP（目的例外）不走代理。
4. **互斥防呆机制（重要）**：为了防止网络冲突，内核配置中实现了互斥防呆：
   * 当您开启 TUN 模式时，系统会自动关闭 TProxy，并将 TProxy 端口自动重置为 0。
   *当您启用 TProxy 模式时，系统会自动关闭 TUN 网卡接管。
   * 如果您的 TProxy 端口为 0 却尝试开启 TProxy 开关，面板会拦截操作并弹出警告提示，请先将 TProxy 端口设定为非 0 端口（例如 7898）。

## 核心功能页面实用技巧

### 1. 仪表盘 (Overview)
* **状态卡片**：展示实时的上传与下载瞬时速度、累计传输流量、内核占用的物理内存、当前的活跃网络连接数及内核版本。
* **外置面板**：仪表盘集成了 MetaCubeXD 与 Zashboard 两个高级外置管理面板的入口。点击即可自动读取面板控制端口与 Secret 并一键跳转登录。
* **自绘流量图**：提供最近 60 秒的自绘流量折线波动图，直观展现网速变化。

### 2. 代理管理 (Proxies)
* **极速模式切换**：不仅可以在配置页切换模式，您还可以在代理页的顶部工具栏，一键直接在 Rule（分流规则）、Global（全局代理）和 Direct（直连）三种核心模式间快速切换。
* **排序与过滤设置**：点击顶部工具栏 of 齿轮图标：
   * **节点排序**：支持按默认、按名称、按延迟（低延迟置顶）或按节点质量评分进行自动排序。对于节点非常多的机场，推荐选择「按延迟」排序，好节点会自动被排在最上方。
   * **正则表达式过滤**：可以输入正则过滤词（例如 `^(?!.*(流量|官网)).*$`），直接过滤并屏蔽掉包含特定无用词汇的节点。

### 3. 规则管理 (Rules)
* **实时规则检索**：当分流发生异常时，可在顶部搜索框输入目标域名，秒级检索当前被哪条分流规则命中。
* **分流规则临时开关**：在 Rules 规则标签页，每条规则右侧都有一个开关。您可以通过开关随时停用或启用某条特定分流规则，不需要修改底层的配置文件。
* **规则提供商更新**：可以在 Providers 标签页对单个规则集进行更新，或者点击「更新全部」。为了保持系统流畅，全部更新功能内置了 3 个并发 Workers 的队列表，在后台排队并发更新。

### 4. 连接追踪 (Connections)
* **双列表机制**：提供「活跃连接」和「已关闭连接」双标签页。活跃连接支持以每秒差分法动态计算每条连接的瞬时网速；已关闭连接历史最多保留最近 100 条。
* **一键阻断与切换修复**：有时您切换了新节点，但浏览器里的视频网页仍然在走老节点甚至卡顿。这是因为旧连接没有断开。此时您可以点击 Connections 页面顶部的「断开所有连接」按钮，强制内核切断所有当前网络握手，浏览器会自动使用新节点建立握手，实现无感平滑迁移。

### 5. 内核配置 (Config)
* **内置 DNS 查询测试**：如果在 Config 页面遇到部分网页打不开或需要排查域名解析，可以在下方的「DNS查询」输入框中输入域名并选择查询类型（如 A 或 AAAA），点击查询，系统会将内核返回的解析 IP 结果平铺展现在下方，并在您查完点击空白处时自动收起手机端键盘，极为便利。
* **缓存一键清理**：当您遇到 DNS 污染或更换了网络节点导致部分网页无法加载，可在此页点击「清空 DNS 缓存」和「清空 FakeIP 缓存」以一键恢复解析。
