# 常见问题解答 (FAQ)

本文汇总了在日常使用 Fluxor 面板过程中，用户经常遇到的问题及其排查与解决办法。

---

### 问题 1: 为什么打开内置的 `MetaCubeXD` / `Zashboard` 外部面板时，提示“无法连接”或需要输入 Host/Secret？
**排查与解决方法**：
1. **外部控制密钥与端口配置**：
   *   外置面板需要通过外部控制器（External Controller）API 与内核通信。
   *   请前往「订阅配置」页面，确认“外部控制端口”和“面板连接密钥 (Secret)”设置正确，并点击底部的「保存并应用」按钮，确保配置已写入底层内核。
2. **手动输入信息**：
   *   如果面板没有自动填充，您可以手动在外部面板连接界面输入：
       *   **Host/IP**：您面板的公网 IP、局域网 IP 或域名（非`127.0.0.1`）。
       *   **Port**：您设置 the 外部控制端口（默认为 `9090`）。
       *   **Secret**：您在订阅设置中定义的密钥。

---

### 问题 2: 面板无法启动底层内核进程，提示启动失败？
**排查与解决方法**：
1. **检查执行权限**：确保您放置在 `/var/apps/Fluxor/target/bin/mi~homo` 路径下的内核二进制文件拥有可执行权限（可以通过终端运行 `chmod +x mi~homo` 赋予权限）。
2. **检查目录读写权限**：确保运行用户对内核工作目录（默认路径为 `/var/apps/Fluxor/shares/Fluxor`）拥有读写权限。
3. **查看内核运行日志**：请前往面板的「日志」页面或者直接在服务器后台查看 `/var/apps/Fluxor/shares/Fluxor/info.log` 日志文件，查找内核报错的具体原因（如端口冲突、配置语法错误等）。

---

### 问题 3: 更新订阅失败，提示“Get ... timeout”或解析出错？
**排查与解决方法**：
1. **核对链接与格式**：请确认您的订阅链接在浏览器中能够正常打开，并且返回的内容包含 `proxies:` 或 `proxy-providers:` 等标准的 cl~a~sh 节点格式。
2. **DNS 解析问题**：
   *   请前往「内核配置」页面，在底部的“DNS 查询”中输入您的订阅域名并点击查询，查看是否能正确解析出 IP。
   *   如果无法解析，说明您的系统 DNS 服务器配置有误，请在系统网络设置中将 DNS服务器修改为公共 DNS（如 `223.5.5.5` 或 `119.29.29.29`）。
3. **目录不可写**：如果您处于“切换模式 (Switch)”，请确保存储各个订阅文件的临时目录 `/var/apps/Fluxor/shares/Fluxor/proxies/` 可写。

---

### 问题 4: TProxy 透明网关富强开启后没有效果，设备无法正常富强？
**排查与解决方法**：
1. **系统环境支持**：TProxy 网关模式强依赖于 Linux 系统的 nftables 工具，请通过命令行执行 `nft --version` 确认系统是否支持 nftables。
2. **核对端口配置**：请确认 TProxy 开启时已绑定了非 0 的局域网监听端口（如 7898）。
3. **本机富强开关**：如果您需要面板主机自身发出的出站流量也走富强，请确认在「例外列表」弹窗中勾选并启用了“本机富强”开关。
4. **清理缓存排查**：如果透明富强偶尔抽风，可能是由于例外列表规则发生变化或防火墙冲突导致，可尝试在「内核配置」页面重启内核以重新下发 nftables 过滤表。

---

### 问题 5: 日志控制台、速度或内存监控显示一片空白？
**排查与解决方法**：
1. **检查内核运行状态**：这些监控依赖于内核的运行。请先确认内核已成功启动并处于“运行中”状态。
2. **排查 WebSocket 连接阻断**：
   *   这些实时图表和数据是通过 WebSocket 升级协议传输的。
   *   如果您通过 Nginx 反向富强访问面板，请确认 Nginx 站点配置中包含了支持 WebSocket 的 Upgrade 头部转发规则。
   *   部分防广告或防追踪的浏览器安全扩展可能会拦截 ws 协议请求，建议尝试开启无痕窗口排除干扰。

---

### 问题 6: 仪表盘上的流量趋势折线图显示一条直线，没有数据波动？
**排查与解决方法**：
1. 检查面板与内核的交互接口是否通畅。请在「配置」中核对内核的 `external-controller` 和 `external-controller-unix` 是否处于默认启用状态。
2. 可以尝试在面板上点击「清空 DNS 缓存」和「清空 FakeIP 缓存」，并重启内核以重置流量传输统计。

---

### 问题 7: 为什么修改了面板参数或者订阅设置，内核却没有发生变化？
**排查与解决方法**：
*  在 Fluxor 中，修改保存订阅或者参数后，**必须点击页面右上方的「保存并应用」按钮**。
*  点击后系统才会重新渲染生成最新的 `config.yaml` 配置文件，并给内核发送配置热重载指令。仅修改输入框内容而不点击保存是不会生效的。
*  如果保存后提示配置已生成，但是内核重载失败，可以检查内核是否未运行，可在配置页尝试手动启动内核，内核启动失败通常为`config.yaml`存在错误配置，订阅保存时错误弹窗通常会提示配置错误出现在`config.yaml`哪一行。

---

### 问题 8: 部分国内网页加载缓慢或打不开，显示域名解析被污染？
**排查与解决方法**：
*   **清空内核缓存**：这通常是由于 DNS 缓存污染导致。请前往「内核配置」页面，在高级维护工具栏中，分别点击「清空 DNS 缓存」和「清空 FakeIP 缓存」。
*   **切换运行模式**：检查您的运行模式是否设置为了 Global（全局富强）或 Direct（直接连接）。如果您希望系统智能识别国内外网站并自动分流，请切换回 Rule（规则分流）模式。
